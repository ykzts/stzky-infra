# エージェント向けガイド

このファイルは、コーディングエージェント向けの実行補助情報をまとめるためのものです。
一般的な開発ルールは `CONTRIBUTING.md` を正本として参照します。

## プロジェクト概要

- このリポジトリは、セルフホストサービスの Docker Compose 構成を管理する
- サービスはディレクトリ単位で独立しており、個別に起動・停止・検証する

## 参照ドキュメント

- 一般ルール: `CONTRIBUTING.md`
- リポジトリ概要: `README.md`
- このファイルには、エージェントが作業時に必要な補足だけを記載する

## 作業フロー

- 変更前に対象サービスの `compose` ファイルと関連ドキュメントを読み、影響範囲を確認する
- 変更は最小差分を優先し、無関係な整形や書式変更を避ける
- YAML 変更後は `yamlfmt .` を実行してフォーマットを揃える
- 可能な範囲で `docker compose config` または対象サービスの起動確認を行う

## 検証コマンド

- 全体フォーマット確認: `yamlfmt .`
- 単体構成確認: `docker compose -f <service>/compose.yaml config`
- テスト設定込み構成確認: `docker compose -f <service>/compose.yaml -f <service>/compose.test.yaml config`

## PR作成時の注意

- 変更内容の要約では、対象サービス・変更理由・想定影響を明記する
- `compose` 設定を変更した場合は、実施した検証内容と未実施項目を明記する
- 一般的なコミットメッセージ規約は `CONTRIBUTING.md` に従う

## 適用範囲と優先順位

- ユーザーの明示的な指示が最優先
- より近い階層の `AGENTS.md` がある場合はそちらを優先
- 競合する一般ルールは `CONTRIBUTING.md` を参照して判断する
